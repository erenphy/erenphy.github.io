在更博、ctf过程中环境配置出现的问题以及解决，不断更新

hexo不显示本地图片

  • 描述

    • 用markdown标记插入图片())，预览没问题，然而在博客中不能显示

  • 解决1

    • 安装插件hexo-asset-image
      npm install https://github.com/CodeFalling/hexo-asset-image --save

    • 并修改根目录配置文件 使得 post_asset_folder: true
      这样在new新的文件.md的同时，生成与该文件同名的文件夹，用于存放图片

    • 但是在npm安装的过程中，出现rollback等error，应该是访问github受限导致的，解决方法暂时不提

  • 解决2

    • 不用markdown语言,而是使用标签插件

      这样debug时，图片可以在网站中显示啦

      npm install

  • 描述

    • 安装失败，显示rollback

  • 解决1

    • 原因是代理配置bug，还是国内外…受限，简单的方法是 npm install -g cnpm --registry=https://registry.npm.taobao.org
      下载淘宝镜像，之后就可以用 cnpm代替npm

  • 解决2

    • 既然可能是github慢的问题，可选择npm安装本地包
      安装本地包，需要cmd转到当前包所在位置，然后 npm install -g 全局安装

      hexo theme config#
